# attributes

Serializers make use of the `@Attribute` decorator to specify which fields should be serialized. When the decorator is called, a single argument is passed, specifying the data type this serializer should be rendering to, like so:

```ts
import { Attribute, DreamSerializer } from '@rvohealth/dream'

export default class UserSerializer<
  DataType extends User,
> extends DreamSerializer<DataType> {
  @Attribute('string')
  public email: string
}
```

## delegate

Attributes can be delegated through another property attached to your data structure, like so:

```ts
import { Attribute, DreamSerializer } from '@rvohealth/dream'

export default class UserSerializer<
  DataType extends { user: User; preferences: any },
> extends DreamSerializer<DataType> {
  @Attribute('string', { delegate: 'user' })
  public email: string
}
```
